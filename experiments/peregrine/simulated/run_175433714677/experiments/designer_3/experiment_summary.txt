THREE PROPOSED MODELS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

┌──────────────────────────────────────────────────────────────────────────────┐
│ MODEL 1: Hierarchical Gamma-Poisson (Random Effects Decomposition)          │
├──────────────────────────────────────────────────────────────────────────────┤
│ Core Idea: Overdispersion = unobserved heterogeneity                        │
│                                                                              │
│ Structure:  C[i] ~ Poisson(λ[i])                                            │
│             λ[i] ~ Gamma(μ[i]×φ, φ)                                         │
│             log(μ[i]) = β₀ + β₁×year[i]                                     │
│                                                                              │
│ Key Parameters: β₀ ≈ 4.3, β₁ ≈ 0.85, φ ≈ 1.5                               │
│                                                                              │
│ Advantage: Explicit random effects, can identify heterogeneous observations  │
│ Risk: May reduce to standard NegBin, computational cost                     │
│                                                                              │
│ Falsification: Abandon if λ[i] shows no pattern or correlates with time     │
│ Runtime: ~5 minutes                                                          │
└──────────────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────────────┐
│ MODEL 2: Student-t Regression on Log-Counts (Robust to Heavy Tails)         │
├──────────────────────────────────────────────────────────────────────────────┤
│ Core Idea: Log-transform counts, use robust regression                      │
│                                                                              │
│ Structure:  y[i] = log(C[i] + 0.5)                                          │
│             y[i] ~ StudentT(ν, μ[i], σ)                                     │
│             μ[i] = β₀ + β₁×year[i]                                          │
│                                                                              │
│ Key Parameters: β₀ ≈ 4.3, β₁ ≈ 0.85, σ ≈ 0.5-1.0, ν ≈ 4-10                │
│                                                                              │
│ Advantage: Robust to outliers, simple interpretation, direct growth rate    │
│ Risk: Ignores count structure, back-transformation may fail                 │
│                                                                              │
│ Falsification: Abandon if ν > 30 (Normal-like) or back-transform fails KS   │
│ Runtime: ~2 minutes                                                          │
└──────────────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────────────┐
│ MODEL 3: Conway-Maxwell-Poisson (Flexible Dispersion, OPTIONAL)             │
├──────────────────────────────────────────────────────────────────────────────┤
│ Core Idea: Don't assume NegBin's variance-mean relationship                 │
│                                                                              │
│ Structure:  C[i] ~ COMPoisson(λ[i], ν)                                      │
│             log(λ[i]) = β₀ + β₁×year[i]                                     │
│             ν controls dispersion (ν=1: Poisson, ν<1: overdispersion)       │
│                                                                              │
│ Key Parameters: β₀ ≈ 4.3, β₁ ≈ 0.85, ν < 0.5 (severe overdispersion)       │
│                                                                              │
│ Advantage: Most flexible dispersion, tests NegBin assumptions               │
│ Risk: Computationally expensive, may reduce to NegBin anyway                │
│                                                                              │
│ Falsification: Abandon if ν ≈ 1 (Poisson) or runtime > 1 hour              │
│ Runtime: ~20-30 minutes (may be longer)                                     │
└──────────────────────────────────────────────────────────────────────────────┘

COMPARISON STRATEGY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Priority Order:  Model 2 (fastest) → Model 1 (moderate) → Model 3 (slowest)

Comparison Metrics:
  1. LOO-CV (ELPD): Primary model selection criterion
  2. Var/Mean Recovery: Must reproduce ≈ 70 in posterior predictive
  3. Convergence: Rhat < 1.01, ESS > 400, no divergences
  4. Interpretability: Can we explain results scientifically?

Decision Rules:
  • ΔELPD > 4:  Clear winner, use that model
  • ΔELPD < 2:  Tied, prefer simpler model
  • All similar: Distributional choice doesn't matter (robust finding!)

FALSIFICATION CRITERIA (EXPLICIT)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Global Red Flags (Abandon ALL models if):
  ✗ All fail Var/Mean check → Pivot to time-varying dispersion
  ✗ All similar LOO-CV → Focus on functional form, not likelihood
  ✗ Parameters inconsistent across models → Data weakly informative
  ✗ High prior-posterior overlap → Need more data or stronger priors

Model-Specific:
  Model 1: Abandon if λ[i] correlates with time (|r| > 0.4)
  Model 2: Abandon if ν > 30 or back-transform fails KS test
  Model 3: Abandon if ν ≈ 1 or computational failure
